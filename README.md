# 中国历史八大皇帝 · 视角 Skill 包

> 「你想蒸馏的下一个员工，何必是同事。」

把中国历史上 8 位最具代表性的皇帝，蒸馏成可直接调用的 Claude Code Skill 文件。

每一个 Skill 不是角色扮演，不是让 AI 模仿某人说话，而是提炼他们**怎么看问题**——心智模型、决策启发式、表达 DNA——打包成可运行的认知框架。

## 每个 Skill 里有什么

每个 `.skill` 文件包含一个 `SKILL.md`，结构如下：

```
心智模型（3—5个）
  ├── 名称 + 一句话描述
  ├── 跨域来源证据（≥2个历史场景）
  ├── 应用方式
  └── 失效条件（何时不适用）

决策启发式（5—7条）
  └── 「若X则Y」格式，有具体历史案例支撑

表达DNA
  ├── 语言风格
  ├── 尺度参照系
  ├── 确定性表达方式
  └── 典型句型

价值观与反模式
  ├── 核心价值观排序
  ├── 绝对不做的事
  └── 已知盲点

诚实边界
  └── 明确标注推断成分、史料偏差、适用范围
```

## 怎么用

### 单视角分析

把问题直接丢给某个皇帝视角：

```
用秦始皇的视角分析：我的 SaaS 产品流失率高，怎么办？
```

秦始皇会问：流失率高的「根」是什么？是产品问题、竞品问题，还是你的获客渠道带来了错误的用户？处理症状（搞召回活动）和根除分叉点（改获客渠道）是两件完全不同的事。

### 多视角对比

同一个问题，用多个框架各分析一遍，看分歧在哪里：

```
同一个问题，分别用秦始皇、汉武帝、唐太宗的视角分析，
给我三个完全不同的切入点
```

- 秦始皇会找「根」（结构问题）
- 汉武帝会找「杠杆」（谁的力量可以被借用）
- 唐太宗会找「人」（谁来做、怎么组合）

三个答案都不是错的，分歧处往往是最有价值的判断点。

### 跨框架互评

让一个皇帝用他的框架评价另一个皇帝的决策：

```
用汉武帝的视角，评价宋太祖杯酒释兵权这个决策
```

## 关于诚实边界

每个 Skill 都内置了「诚实边界」板块。

蒸馏历史人物和蒸馏在世人物有一个根本区别：在世人物有访谈、播客、社交媒体等第一手话语记录；历史人物的材料几乎全是「他做了什么」，需要反推「他怎么想」，推断成分不可避免。

此外，史料本身存在叙事倾向：

- 《史记》是汉代人写的秦代史
- 《贞观政要》经过了理想化处理
- 《清实录》对康熙不利的内容有删减
- 《蒙古秘史》是蒙古人自己写的

所以每个 Skill 都会明确标注：材料来源、置信度等级、哪个阶段的他更可信、哪些结论是推断。

**一个不告诉你局限在哪的 Skill，不值得信任。**

---

## 置信度说明

| 等级 | 含义 | 对应人物 |
|------|------|---------|
| `high` | 一手史料充分，行为记录翔实，学界研究成熟 | 唐太宗、康熙 |
| `medium-high` | 核心材料可信，部分细节依赖推断 | 秦始皇、汉武帝、朱元璋、隋文帝、宋太祖 |
| `medium` | 材料来源存在偏差，推断成分较高 | 成吉思汗 |

## 这套 Skill 是怎么做出来的

基于 [nuwa-skill](https://github.com/alchaincyf/nuwa-skill)（花叔开源）的蒸馏方法论，并做了以下优化：

- **信息量预判机制**，调研开始前先探一手资料量，不足时自动降为草稿模式并打水印，而不是走完整流程交出一个看起来完整实际上靠臆测填充的 Skill
- **串行降级模式**，在没有子智能体的环境（如 Claude.ai）中自动切换为单 Agent 串行执行，不再是「装了用不了」
- **结构化验证评分表**，把「方向是否一致」这种主观判断变成 0—2 分的逐题评分，通过门槛 ≥4/6 分，低于此回炉重做
- **输出 metadata 字段**，每个 Skill 头部标注蒸馏日期、来源数量、置信度、模式、建议复评日期

蒸馏三重验证标准（来自 nuwa-skill，完整保留）：

一个心智模型要被收录，必须同时满足：跨域重现（在 2 个以上不同领域出现过）、有预测力（能推断这个人对新问题的立场）、有排他性（不是所有聪明人都会这么想）。

## 关于成吉思汗的特别说明

成吉思汗的征服导致了历史上规模最大的人口死亡事件之一，中亚、华北、波斯的人口在蒙古征服后数十年内锐减超过 50%。

这个 Skill 提炼的是他的认知框架，不是对他行为的背书。使用时请保持清醒。
